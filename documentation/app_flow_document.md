# App Flow Document

## Onboarding and Sign-In/Sign-Up

When a new developer discovers the React-Starter repository, they begin by cloning the project from GitHub and opening the folder in their preferred terminal and VS Code editor. Before running any code, they create a `.env` file at the root of the project, copying keys for the Node.js API endpoint, the Cloudflare D1 database URL, and the Claude AI API token. To enable Cloudflare operations, they run `wrangler login` and follow the browser-based OAuth flow to authorize the project. For AI-assisted development, they run `claude login` and enter their API credentials when prompted. With accounts linked, they install all dependencies with `pnpm install` (or `npm install` depending on the chosen package manager). If the developer needs to switch accounts or revoke access, they can use `wrangler logout` or `claude logout` and repeat the login steps to sign in again. Password resets or key rotations are handled through each service’s web dashboard following their standard recovery flows.  

## Main Dashboard or Home Page

Once the developer runs the combined development command via `pnpm run dev`, two local servers start: the React frontend at http://localhost:3000 and the Node.js API at http://localhost:4000. Opening http://localhost:3000 displays the starter homepage, which features a header showing the application name, a simple navigation bar with links to Home, Sample API Test, and Project Settings, and a welcome message explaining that this is a scaffolded environment. The navigation bar lets the developer switch instantly between the core pages. At the bottom of the page, there is a button labeled “Call Sample API,” which triggers a fetch request to `/api/hello` on the backend. The response, a JSON object with a greeting message, appears in a console-style output box below the button. This home view is the central point from which all UI pages are accessed.  

## Detailed Feature Flows and Page Transitions

On the Sample API Test page, the developer sees a list of predefined endpoints and an input field to send custom payloads. When they enter a JSON snippet and hit “Send,” the frontend makes a POST request to `/api/test` and displays the response object in the same console-style output box. Navigating back to the Home link updates the address bar to `/` and re-renders the welcome message without a full page reload, thanks to React Router’s client-side routing. If the developer wishes to extend the API, they open the `apps/api/src/lib` folder, create a new file for their endpoint logic, and register the route in the Express (or NestJS) router. After saving changes, they see the API server automatically restart and can immediately test the new endpoint in the Sample API Test page.  

For backend database changes, the developer edits the schema file in the Terraform directory under `infrastructure/`. They then run `terraform init` to configure the state backend, and `terraform plan` to preview changes. To apply the updates, they run `terraform apply`, which provisions or updates the Cloudflare D1 instance. If they need AI guidance on schema best practices, they use the Claude command defined in `.claude/commands/migrate-to-d1.md` by running `claude run migrate-to-d1.md`. The CLI outputs step-by-step instructions, which the developer follows in the terminal. Once the database is provisioned, the developer can invoke migration scripts via the backend CLI or manual SQL files to sync the schema.  

## Settings and Account Management

To adjust personal preferences or change environment variables, the developer edits the `.env` file at the project root. They might update `API_URL` to point at a staging server or rotate the `D1_DATABASE_URL` after rotating keys in the Cloudflare dashboard. If they want to customize their Git hooks, they open `.husky/pre-commit`, modify lint or test commands, and run `husky install` to reinitialize the hooks. In VS Code, they can override settings by editing `.vscode/settings.json` to add themes or formatter rules. After saving, the editor prompts them to reload the window, ensuring the updated settings take effect. When they finish, they usually return to the main development flow by running `pnpm run dev` again, which brings up the homepage and API servers.  

## Error States and Alternate Paths

If the developer omits required environment variables, the frontend or backend server fails to start and logs an error identifying the missing keys. They are prompted to populate the `.env` file and restart the dev command. In the event of a Terraform misconfiguration, running `terraform plan` outputs syntax or permission errors. The developer corrects the HCL file and reruns the plan. Should the Cloudflare D1 database reject a migration because of an existing table or schema mismatch, the migration script exits with an error code. The developer then manually inspect the SQL, drop or rename the conflicting resource, and retry the migration. On Windows machines, if the Husky hook fails under PowerShell, the developer can switch to Git Bash or configure `core.hooksPath` in the global Git config. When Claude commands encounter authentication failures, the CLI prints a clear message and suggests running `claude logout` followed by `claude login` to refresh credentials.  

## Conclusion and Overall App Journey

From cloning the repository to pushing new features, the developer’s journey begins with setting up accounts, logging in to Cloudflare and Claude, and installing the workspace dependencies. They then launch both the React frontend and the Node.js API, explore the sample homepage and API test page, and extend the code by adding new routes or components. Infrastructure provisioning and database migrations are managed through Terraform and optional AI-assisted commands. Throughout the process, Git hooks and VS Code settings enforce code quality. Whether fixing a missing environment variable or resolving a database migration conflict, the developer always returns to the familiar local home and API test pages, iterating quickly until the new feature is production-ready and merged into the main branch.
